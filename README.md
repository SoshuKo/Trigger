## v20 戦闘ルーチン・武器動作・詳細ログ

### NPCの交戦判断

- 射手・銃手・狙撃手は、敵本体ではなく武器の適正距離を保てる交戦地点を短時間固定して移動します。
- 標的は原則1.5～2.5秒維持し、被弾、見失い、明確に有利な敵、オペレーター指示がある場合だけ即変更します。
- スタック脱出後は直前の経路を短時間避け、同じ場所へ戻る往復を抑えます。
- ランダムチームは前衛・射撃・防御／支援の偏りを抑えた役割構成になります。
- 砂漠の安全地点はトリオン22%未満から検討し、短時間で離脱して同じ地点へ20秒以上戻りません。
- NPCオペレーターの指示は状況が変わらない限り更新せず、通常10～18秒の不規則な間隔です。

### 武器動作

- シューターはトリオンキューブを展開してから発射します。準備中と発射直後は、同じ手の別シュータースロットへ連続で切り替えて撃てません。左右の手は独立しています。
- ガンナーは銃種と弾種ごとの弾倉を持ち、残弾0でリロードします。HUDへ残弾とリロード進行を表示します。
- レイガストは `Shift` を押しながら使用すると盾になります。
- 弧月・スコーピオン・レイガストは短い間隔で連撃でき、最大段数または入力終了後にクールダウンが発生します。
- 狙撃銃の適正距離を広げ、スコープの先読み距離を大幅に延長しました。
- 射手アステロイドは基礎威力を約8%抑えています。

### 詳細ログ

戦闘記録は `schemaVersion 20` です。JSONには次の情報を追加しています。

- 射手のキューブ展開開始・発射・中断・同じ手の切替拒否
- 銃ごとの発砲数、空撃ち、リロード開始・完了・所要時間
- 近接連撃の各段、終了理由、終了後クールダウン
- 命中距離、適正距離内／近すぎる／遠すぎる命中
- NPCの標的変更理由、標的維持回数、前進・後退・維持・横移動時間

- 役割別平均、武器別集計、イベント種別ごとの発生回数

CSVにもAI距離行動、射手準備、リロード、近接連撃、レイガスト防御の列を追加しています。

## v19 ジャスト判定・適正距離・スコープ

### 弧月／スコーピオンのジャスト判定

- 斬撃直後に約0.13～0.20秒のジャスト判定が発生します。
- 判定中、正面から来た直接攻撃や弾を切るとダメージを受けません。
- 成功した弧月／スコーピオン枠は、残っているクールダウンが半分になります。
- 爆発、毒、地形ダメージなどの範囲攻撃は切れません。
- スコーピオンの長刃でもジャスト判定が発生します。

### 射撃トリガーの適正距離

シューター、ガンナー、スナイパーの弾にはトリガーごとの適正距離があります。命中地点が近すぎる、または遠すぎる場合はダメージが50%になります。NPCにも同じ補正が適用され、NPCは従来の役割別距離判断を使って適正距離を維持します。

### ガンナー／スナイパーのスコープ

- ガンナーまたはスナイパーを選択中に `R` でスコープを切り替えます。
- 固定半径を円周移動する方式を廃止し、プレイヤーから照準線を直線的に伸縮する方式へ変更しました。
- PCはマウス左右で方向、上下またはホイールで近距離／遠距離をゆっくり調整します。
- モバイルは右パッドの左右で方向、上下で照準距離を調整します。
- スコープ中は照準地点に合わせてカメラが滑らかに先行します。
- 敵へ照準が合うと距離を表示します。
- 適正距離では照準が緑、近すぎる／遠すぎる場合は橙になります。
- モバイルでは `SCOPE` ボタンを使用します。

## v18 NPC思考・防衛戦更新

### NPCの移動判断

個人戦、チーム戦、防衛戦のすべてで同じ進路安定化処理を使用します。

- 一度決めた前進・後退・横移動を短時間維持し、距離の境界で行ったり来たりする動きを抑制
- 直前の移動方向と目的地を記憶し、急な180度反転が続いた場合は横方向の脱出地点を選択
- 一定時間の移動量だけでなく、目的地へ近づいているかも使って詰まりを判定
- 壁の迂回地点と探索地点の履歴を残し、直前に失敗した経路を選び直しにくくする
- 味方同士の重なりを避ける分離移動を追加
- 敵を見失った探索、砂漠の退避、フラッグ修復、防衛戦の侵攻兵にも同じ処理を適用
- 壁の破壊は、迂回と脱出を試しても進めない場合だけ実行

### 防衛戦の調整

- 通常侵攻兵とブラックトリガーの耐久・攻撃力・移動速度を全体的に低下
- 通常ラウンドの出現数とラービット出現率を低下
- イルガーの爆撃間隔を延長し、自爆範囲と威力を低下
- ラービットのキューブ化時間とトリオン吸収量を低下
- ブラックトリガーの状態異常時間とフラッグへのダメージ倍率を低下
- フラッグ耐久を大幅に増加し、22%の装甲と非戦闘時の自然修復を追加
- ラウンドクリア時のフラッグ回復、隊員のトリオン回復、防衛資材補給を増加

### フラッグ防衛設備

防衛戦では画面右下の「フラッグ防衛」パネルから、防衛資材を使って設備を自動配置できます。戦闘員と盤面外オペレーターの両方が使用できます。

| 設備 | 役割 |
|---|---|
| 防壁 | 3枚組の壁をフラッグ周囲へ展開し、侵攻経路を遅らせます。 |
| 固定トラップ | 接近した侵攻兵へ範囲攻撃を行います。 |
| 固定砲台 | 周囲の侵攻兵を自動射撃します。 |
| 囮ビーコン | 一部の侵攻兵を短時間フラッグから引き離します。 |

防衛資材は開始時に支給され、敵撃破とラウンドクリアで補充されます。NPCも安全な準備時間に不足している設備を展開します。設備には耐久と制限時間があり、敵の攻撃でも破壊されます。オンライン戦では設備本体、耐久、残り資材をホストから同期します。

## 試合

### 個人戦

CPUを3～19人まで設定できます。撃破とトリオン粒子の回収で得点します。

### チーム戦

- 参加チーム：2～4チーム
- 1チームの戦闘員：1～4人
- 初期値：2チーム・各3人
- 各チームに盤面外のオペレーター1人

出撃設定では、次の参加方法を選べます。

- **戦闘員**：自チームの戦闘員枠を1人分使用して出撃します。
- **オペレーター**：盤面には出現せず、戦闘員数にも含まれません。作戦画面を移動しながらNPCへ戦術指示と支援を送ります。
- **観戦**：盤面には出現せず、戦闘員数にも含まれません。各隊員の視点を切り替えて観戦します。

オペレーターは、集中攻撃、地点移動、待機、集合、防衛、左右の側面展開、退避、分散展開、地域制圧、消耗した敵の追撃、護衛、設備起動を指示できます。NPCは受信した命令を通常のAI判断より優先します。

オペレーター支援として、広域索敵、トリオン補給、照明ビーコン、緊急バリケード、機動支援、囮ビーコンを使用できます。位置指定型の支援は作戦画面上をクリック／タップして発動します。

### 市街

### 砂漠

- **古代要塞**：日陰になった内部に、バリケード、固定トラップ、固定砲台、破壊可能な松明があります。
- **崖**：表示された矢印方向へ進む一方通行の経路です。逆方向への移動は押し戻されます。
- **流砂**：移動速度が落ち、流れと中心方向へ継続的に引き込まれます。
- **砂丘**：広範囲に広がり、移動速度が少し低下します。
- **オアシス**：水中では移動が遅くなりますが、昼間の環境負荷を解除します。
- **松明**：各所に点在する破壊可能な火と光源です。夜間の視界と環境負荷の解除に利用できます。
- **ガス田**：爆発を伴う攻撃が近くで発生すると大爆発します。爆発後は一定時間でガスが戻ります。

砂漠では雨は降りません。昼間は日陰またはオアシスの外でトリオン消費が38%増加し、夜間は火から離れると28%増加します。NPCもトリオンが少なくなると日陰、オアシス、火を目指します。

背景は640×640の区画ごとに生成・再利用し、表示中の範囲だけ描画します。

## 環境・オブジェクト更新

- エスクード、ワイヤー、設置メテオラ、ダミービーコン、スイッチボックスなどの設置物は、種類ごとの制限時間で消滅します。
- オペレーター設備のバリケードは約48秒、固定砲台は約62秒、固定トラップは約70秒で停止します。
- 破壊された林、ビル壁、橋は一定時間後に復旧します。復旧地点に隊員がいる場合は、安全になるまで復旧が延期されます。
- 破壊された固定設備は約45～70秒、光源は約38～62秒で復旧します。
- 夜は自分、トリオン粒子、破壊可能な照明の周囲だけ視界が開けます。照明は道路や広場付近に自動生成されます。
- 雨天では地面の摩擦が下がり、進行方向を変えても慣性が残ります。道路と広場は特に滑りやすくなります。
- 戦闘記録形式は `schemaVersion 20` です。参加役割、チーム数、選択マップ、オペレーターの指示・支援回数、破壊・復旧・デスポーン件数、ガス田爆発回数も記録します。

## NPCの砂漠行動修正

砂漠でNPCが日陰・オアシス・松明の周囲に長時間停滞する問題を修正しました。

- バッグワーム、バッグワームタグ、カメレオンを状況に応じて自動解除
- 遮蔽トリガー解除後に4～7秒の再使用待ち時間を追加
- 砂漠の負荷中は、十分なトリオンがある場合だけ遮蔽トリガーを使用
- 安全地点へ向かう条件をトリオン19%未満まで引き下げ
- 敵が近い、攻撃を受けた、弾が迫っている、オペレーター命令がある場合は退避行動を中止
- 安全地点への退避は最大数秒で終了
- 敵を見失ったNPCは停止せず、周辺を索敵移動
- トリオン22%未満では継続消費トリガーを強制解除


## NPC経路探索・ダミービーコン認識の改善

- NPCは進行方向を塞ぐ壁を検出すると、壁の四隅から最短の迂回点を選びます。
- 壁越しの敵には攻撃せず、射線が通る位置まで移動します。
- 壁破壊は、複数の迂回候補がすべて使えず、長時間動けない場合だけの最終手段です。
- ダミービーコンは距離、観察時間、移動の不自然さ、被弾後の反応、周囲にいる実在敵から識別します。
- 一度ダミービーコンだと識別したNPCは、そのビーコンを再び標的にしません。
- チーム戦では識別結果を同じ隊のNPCへ共有します。
- 戦闘記録へ壁回避回数、壁破壊への切替回数、ダミービーコン識別回数を追加しました。


## 自主ベイルアウトと砂漠退避の改善

- NPCは体力が著しく低下すると、自主ベイルアウトを判断することがあります。
- 判断基準は役割ごとに異なり、狙撃手・工作手はやや早く、攻撃手・重装手はより瀕死になるまで粘ります。
- 最後の被弾、盾で防いだ攻撃、接近中の飛来弾から約4.2秒以内は自主ベイルアウトできません。
- 敵が近距離にいる場合も自主ベイルアウトを選びません。
- 自主ベイルアウトでは相手にキル、撃破点、アシスト点は入りません。
- プレイヤーの自主ベイルアウトにも同じ攻撃直後の制限を適用しました。
- 日陰、オアシス、火へ向かう退避判断をさらに厳しくし、通常時はトリオン16.5%未満で敵が遠い場合だけ使用します。
- NPCは避難先の中心ではなく、安全範囲の手前側へ入ります。同じ場所への密集も避けます。
- 安全範囲へ到着した後は短時間で離脱し、一定時間は同じ避難先へ戻りません。
- 戦闘記録へNPC自主ベイルアウト回数、安全地点への訪問回数、離脱回数を追加しました。

## v12 防衛戦

出撃設定から「防衛戦」を選択できます。防衛戦では1隊だけが参加し、隊の戦闘員数を1～4人から選択します。戦闘員として出撃する場合はプレイヤーが1枠を使用し、オペレーターまたは観戦を選んだ場合は設定人数すべてがNPC戦闘員になります。

### 基本ルール

- マップ中央に防衛フラッグが1基出現します。
- 敵はラウンド制で侵攻し、全滅させると次ラウンドへ進みます。
- 5ラウンドごとに敵の耐久・攻撃力・速度が強化されます。
- 5、10、15、20、25ラウンドではブラックトリガーのボスが1体出現します。その後は同じ順番で強化個体が再登場します。
- フラッグの耐久が0になると防衛終了です。
- 戦闘員はフラッグ付近でFキー、またはモバイルのFLAGボタンを押すと、自分のトリオンを消費してフラッグを回復します。
- NPCも安全が確保され、フラッグ耐久が低いときにトリオンを注入します。
- オペレーターは専用支援「フラッグ緊急修復」を使用できます。
- フラッグには22%の装甲があり、攻撃を受けていない間は少しずつ自然修復します。
- 防衛資材を使って、防壁、固定トラップ、固定砲台、囮ビーコンをフラッグ周囲へ展開できます。
- 敵撃破とラウンドクリアで防衛資材が補充されます。

### 通常侵攻兵

- **モールモッド**：高速で接近し、硬い脚部ブレードで戦闘員やフラッグを攻撃します。
- **イルガー**：上空から爆撃します。耐久が減ると装甲を強化し、フラッグや人が集まる地点へ自爆降下します。背面攻撃には迎撃爆発で反応します。
- **ラービット**：高耐久の捕獲型です。近接攻撃を受けた戦闘員は一時的にキューブ化され、移動とトリガー操作を封じられます。正面装甲はダメージを軽減します。

### ブラックトリガーボス

1. **風刃**：複数の遠隔斬撃と、遮蔽物を経由するフラッグ狙いの斬撃を放ちます。
2. **印**：錨・響・門・盾・鎖・弾をランダムに使用し、一定回数ごとに多重印として強化します。
3. **アレクトール**：着弾地点を予告する弾を展開し、命中した戦闘員を一時的にキューブ化します。
4. **ボルボロス**：固体・液体・気体を切り替えます。液体・気体中は攻撃が通りにくく、気体時は毒ガスを展開します。
5. **オルガノン**：本体を中心に複数の円軌道刃を走らせ、戦闘員とフラッグをまとめて攻撃します。

すべての大技には攻撃範囲の予告表示があり、原作上の特徴をゲームとして回避可能な攻撃へ調整しています。

### 戦闘記録

戦闘記録は `schemaVersion 20` です。到達ラウンド、強化段階、撃破した通常敵・ボス数、フラッグ耐久・回復量、現在の侵攻敵を保存します。


## v15 オンライン対戦の安定化

- チーム戦では各TEAMを設定人数までCPUで補充します。フレンドが参加した枠だけCPUと置き換わります。
- 防衛戦では設定した防衛隊人数までCPUを補充します。オペレーターと観戦者は戦闘員数に含みません。
- 個人戦では設定された総参加枠を基準に、オンライン参加者以外をCPUで補充します。
- TEAM 3・TEAM 4を選んだオペレーターや観戦者がいる場合も、必要なチーム数を自動的に確保します。
- ホストが開始するとオンラインロビーを閉じ、全端末を試合画面へ移動します。
- Realtimeの開始通知を取り逃した場合も、ルームの `playing` 状態から試合開始を復元します。
- 初期マップと最初のスナップショットを受信するまでは、仮の盤面を表示せず同期画面を表示します。
- 古いスナップショットを破棄し、隊員の座標・速度・向きを補間して描画します。
- モバイルのアドレスバー表示などで画面サイズが変わっても、カメラ中心を維持します。
- 同期が完了しない場合は、クライアントがホストへ定期的に再同期を要求します。


## v16 オンラインゲストの軽量化

- ホストから常時送るスナップショットを、隊員・弾・必要な戦闘演出だけに縮小しました。
- 装備、能力、隊章などの固定情報は初期同期時だけ送信します。
- 回収物、ワイヤー、罠、ビーコン、壁、設備、照明の状態は約1秒ごとの低頻度同期へ分離しました。
- ゲストは同時に届いた古いスナップショットを処理せず、最新の1件だけを反映します。
- 弾は受信間隔の間も速度からローカル補間し、演出時間も端末側で進めます。
- ゲストのレーダーとHUD更新頻度を下げ、Canvas描画を最大45fps、低性能・モバイル端末では最大30fpsに制限します。
- ゲストの描画解像度、地形キャッシュ、影、雨粒、画面外エフェクトを自動的に軽量化します。
- オンラインの標準同期頻度を5Hzへ調整しました。参加人数が増えるとさらに自動で下がります。
